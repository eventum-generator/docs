The Overview page is the first thing you see when you open Studio. It serves as a central hub — showing key information about your Eventum installation, quick actions for common tasks, navigation to all major areas, and a list of recently used projects.

## Header

The top section displays:

- **Eventum Studio** — the application name with a version badge.
- **Version** — the current Eventum version (e.g. 2.6.1).
- **Platform** — the OS, kernel version, and architecture of the host machine.

## Quick actions

Two action cards let you jump straight into common workflows:

| Action | Description |
|--------|-------------|
| **New Project** | Create a new generator project — set up event production, add scheduling, and configure where your synthetic data goes. |
| **Browse Eventum Hub** | Explore ready-to-use generators for popular data sources — deploy production-grade synthetic events in seconds. |

## Navigation cards

Below the quick actions, three cards link to the main areas of Studio:

| Card | Description |
|------|-------------|
| **Projects** | Manage projects in one place. Open any project to visually design its complete plugin pipeline. |
| **Instances** | See everything right now. Each instance shows its performance metrics, structured logs, and full lifecycle controls. |
| **Scenarios** | Manage generator workflows — coordinate instances with global state, data flow control, and lifecycle controls. |

## Recent projects

The bottom section lists recently used generator projects, ordered by last activity. Each entry shows the project name and the time since it was last accessed (e.g. "11 minutes ago"). Click any project to open it in the [project editor](/docs/studio/projects).
