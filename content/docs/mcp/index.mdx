The Eventum **MCP server** lets an AI agent — Claude Code, Cursor, GitHub Copilot, or any other [Model Context Protocol](https://modelcontextprotocol.io) client — build and run Eventum generators directly, grounded on the real engine instead of guesswork.

You stay in your editor or chat. You say *"give me a week of Apache access logs with a traffic spike at noon, mostly 200s but a burst of 500s during the spike"*, and the agent writes the files, validates and previews them against the real pipeline, and fixes them until the output is right. See [how it works](/docs/mcp/how-it-works) for the full loop.

## No model required

Eventum embeds **no language model**, no API keys, and no per-token cost. Your agent already has a model; the MCP server gives that model an exact, structured view of Eventum:

- the **schema** of every input, event, and output plugin,
- the **template helpers** available inside events (always current — generated from the live code),
- **worked examples** it can copy from,
- and **real-engine feedback** — `validate` and `preview` run the actual pipeline, so the agent sees the same errors and the same output you would.
